WeViews

Tutorial

Install

  • Download the distribution from here (zip, tarball).
  • Add only the "WeViews" subdirectory to your project.
    Do not add the "WeViewsDemo" subdirectory to your project.

Core Concepts

  • All UIViews have a "natural" size. This is the size returned by [UIView sizeThatFits:].
    For an UIImageView, its the size of its image. For a (non-wrapping) UILabel, its the size of its text in its font.
  • FPanels re-layout their contents whenever a) their size is changed with [setFrame:] or [setBounds:] or b) [FPanel layoutContents] is invoked.

Demo App

  • The Demo app allows you to build and tinker with user interfaces interactively. Playing with it is a good way to get a feel for how the library works.
  • To run the demo, open the WeViewsDemo Xcode project.
  • The Demo requires iOS 3.2+ and is best run in the iPad Simulator.
  • Add views to the canvas using the "Add Views" sidebar. New views are added to the currently selected view or layer.
  • Select views by clicking on them in the canvas or by selecting them in the "View Hierarchy" sidebar.
  • You can inspect and modify view & layer properties using the "Selection" sidebar.
  • One useful feature of the Demo app is that it can generate example code for any UI you have built. Select any view and use the "Generate Code" link in the "Selection" sidebar.

    WePanels group their subviews in layers. Most WePanels will only have a single layer.
    Each layer has its own layout strategy, alignment, margins, spacing, etc.
    In this example, a WePanel has a background image in one layer and a group of buttons is a second layer. The background layer uses a fill layout with small (5 point) margins. The buttons layer uses a horizontal layout with a large (20 point) bottom margin and bottom alignment.
    You shouldn't need to deal much with the WePanelLayer class. Layers are easily created and configured anonymously.

The code doesn't need to explicitly refer to the WePanelLayer class, retain it or assign it to a variable.
  • Grid Layouts
    The "Natural Grid" layout lays out its views at their natural size. Each column has the width of its widest element; each row has the height of its tallest element.
    The example on the right is left- and top-aligned. Note how the alignment effects the rows and columns AND the views within their grid cells.
    The "Spacing Grid" layout lays out its views at their natural size. Extra space is divided evenly between the spacing between rows and columns.
    The "Fill Grid" layout lays out its views at their natural size. Extra space is divided evenly between the rows and columns.
    Stretching views are stretched to fill their cell. Non-stretching views are aligned within their cell. The example on the right is left- and top-aligned.
    The "Even Fill Grid" layout ignores the natural size of its views. All rows have equal height and all columns have equal width.
    Use the "cellSizeHint" property to control the natural size of the grid.
    As with the "Fill Grid", Stretching views are stretched to fill their cell. Non-stretching views are aligned within their cell. The example on the right is left- and top-aligned.
  • More on Grids
    With all grids, you can set either the row count or the column count, but not both. If you set the row count, the column count is (total number of views in layer / row count).
    Grid layouts divide their space into cells. Stretching views will be stretched to fill their cell. Non-stretching views will be aligned within their cell using the layer's alignment properties.
