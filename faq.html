WeViews

FAQ

  • When does WePanel layout its contents?
    When do I need to manually trigger layout?
    Whenever you change the size of a WePanel using [WePanel setFrame:] or [WePanel setBounds:], the WePanel triggers layout by invoking its [WePanel layoutContents] method. Calls to [WePanel setFrame:] or [WePanel setBounds:] that do not change the WePanel's size do not trigger layout.
    [WePanel layoutContents] can be invoked whenever layout is necessary, such as:
    • After adding or removing a view or layer.
    • After changing a layer's layout strategy.
    • After changing a layout properties, such as margins, spacing, alignment, etc.
    If a WePanel contains subviews which are themselves WePanels, resizing it will automatically cause these subpanels to themselves layout if their size is changed. Therefore, it is only necessary to manually layout the root WePanel.
  • Can I mix and match WeViews with other UIViews?
    Yes. WePanel, for example, can layout any properly implemented UIView (ie. that has implemented [UIView sizeThatFits:]). Likewise, WeView classes can be safely used like any other UIView.
  • What's the difference between "fill" and "stretch"?
    Some WePanel layouts try to lay out views at their "natural" size. This is the size returned by [UIView sizeThatFits:]. Other WePanel layouts try to stretch subviews to fill the panel's frame. We call these "Fill" layouts.
    UIViews and most WeViews do not "stretch" by default. WeViews have a stretchWeight property that determines whether or not they stretch. A stretchWeight of zero indicates that the view does not stretch.
    Fill views try to divide their extra space proportionally based on stretchWeight. Subviews with larger stretchWeight receive more space.
  • My views aren't laying out right.
    How do I debug the layout process?
    If you set the debugLayout property of a WePanel to YES, it will dump debugging info to the console. It also draws a checked box over its bounds.
    You can also recreate your problem in the Demo app. The Demo app let's you modify and inspect view hierarchies interactively.
  • Why isn't my view showing up?
    Many UIViews (such as a UIScrollView or an empty WePanel) have a natural size of zero. If it is an empty WePanel, try adding a layer with some subviews. Or, if the view itself is layer out in a fill layout, try giving it a non-zero stretch weight.
  • Does WePanel support sub-point layouts?
    No. WePanels only support laying out subviews with integer locations and sizes.
  • Can I define a custom layout strategy?
    Yes. Subclass WePanelLayout. [WePanelLayout mode] should return LAYOUT_MODE_CUSTOM. [WePanelLayout sizeThatFits:layer:] should return the minimum "natural" size of the layer's contents. [WePanelLayout layoutContents:layer:] should lay out the layer's contents.
